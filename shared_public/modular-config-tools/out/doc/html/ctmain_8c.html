Prototype description
======================
Prototype for specific config tool main function.

Note
The function implementation has to be provided application specific.
Parameters
argcArgument count, directly forwarded from outer main function.
argvArgument vector, directly forwarded from outer main function.
pstOptionsOption structure pointer with common and specific application options.
pstResourcesResource structure pointer with common and specific application resources.
Returns
Application exit code, CTUTIL_EXIT_SUCCESS on success, error code otherwise. The provided exit code should be positive and never a forward from a status code. The provided exit code should be between CTUTIL_EXIT_MIN and CTUTIL_EXIT_MAX. The provided exit code should honor the common exit code defines and avoid any conflict.

Function to initialize config tool specific resources (for example implementation see HelloWorld example config tool).

Prototype description
======================
Prototype to initialize application specific resources.

Note
The function implementation has to be provided application specific.
An eventually set pointer for a specific resources structure have to be released by ctparts_ReleaseSpecificResources.
Parameters
pstContextApplication specific context. If unused, NULL is allowed.
ppstResourcesPointer to resource structure pointer to be equipped with application specific resource data or pointers.
Returns
CTUTIL_SUCCESS in case of successfully acquired resources, an error code otherwise.
Precondition
Application specific resource pointer ctutil_ResourcesSpecific_t points to NULL.
Postcondition
Application specific resource pointer ctutil_ResourcesSpecific_t points to NULL or a valid structure in case of success. Application specific resource pointer ctutil_ResourcesSpecific_t points to NULL in case of an error.
